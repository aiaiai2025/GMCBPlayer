# Unexpected Stories — aandGo Stem Player

## Project Overview

A browser-based stem player for the musical "Unexpected Stories." Built as a single-page GitHub Pages app. Musical directors and rehearsal pianists can play backing tracks with a mixable vocal guide, control tempo and pitch, and manage the show's set list.

**GitHub repo:** https://github.com/aiaiai2025/UnexpectedStoriesaandGo  
**GitHub Pages URL:** https://aiaiai2025.github.io/UnexpectedStoriesaandGo/

## songs.json Format

Generated automatically. Paths are always prefixed `audio/`. Supports a special multi-part structure for vamp songs:

```json
{
  "showTitle": "Unexpected Stories",
  "songs": [
    {
      "title": "01 - Our Turn Now",
      "instr": "audio/01 - Our Turn Now - FINAL (Instrumental).mp3",
      "vocal": "audio/01 - Our Turn Now - FINAL (Vocals).mp3"
    },
    {
      "title": "07 - Peddler Song - Reprise",
      "instr": "audio/07 - Peddler Song PT1 - Reprise (Instrumental).mp3",
      "vocal": "audio/07 - Peddler Song PT1 - Reprise (Vocal).mp3",
      "vamp": {
        "instr": "audio/07 - Peddler Song VAMP - Reprise (Instrumental).mp3",
        "vocal": "audio/07 - Peddler Song VAMP - Reprise (Vocal).mp3"
      },
      "postVamp": {
        "instr": "audio/07 - Peddler Song PT3 - Reprise (Instrumental).mp3",
        "vocal": "audio/07 - Peddler Song PT3 - Reprise (Vocal).mp3"
      }
    }
  ]
}
```

**Vamp detection in generate_songs.py:** Files containing `PT1`, `VAMP`, or `PT3` in their names are automatically grouped into a single song entry with nested vamp/postVamp sections.

## Player Features (index.html — v2.15)

### Core audio engine
- **SoundTouch** library embedded inline — runs in main thread (no AudioWorklet needed), works with `file://` and GitHub Pages
- Dual stem playback: Instrumental + Vocal Guide mixed in real time
- Vocal volume slider (Off / Low / Mid / Full presets)
- Tempo adjust: 75–125% (SoundTouch time-stretch, pitch-preserving)
- Key/Pitch adjust: ±5 semitones

### Vamp playback engine
Song 07 (Peddler Song Reprise) has three parts:
1. **PT1** plays once through
2. Automatically transitions to **VAMP** which loops indefinitely with crossfade
3. **Finish Loop** button — queues PT3 to start at the end of the current loop iteration (clean transition)
4. **Go Now** button — crossfades immediately into PT3
5. PT3 plays once to end

### Modes (four-button toggle in header)
- **Rehearsal** (default): Full controls. No auto-advance — manual navigation only.
- **Show**: Song ends → auto-loads next → waits for Space/Play. For live performance with a human triggering each cue.
- **Auto**: Song ends → auto-loads and immediately plays next. For dance classes, run-throughs, unattended playback.
- **Show W/Cues**: Like Show mode but includes music cues (see below) interleaved with songs in natural sort order (01, 01a, 01b, 02…). Cues are hidden from all other modes and playlists.

### Music Cues (v2.14)
- Files ending in `(Cue).mp3` or `-Cue.mp3` are detected by `generate_songs.py` and stored as `{ "type": "cue", "src": "..." }` in songs.json
- Named with alphanumeric prefixes (`01a`, `01b`) so they sort between songs naturally
- **Simplified UI**: no vocal controls, no pitch — only Tempo, Loop toggle, and Fade Out button
- **Loop off by default**: cue plays once and stops; board op enables loop if the scene change needs extra time
- **Loop toggle (⟳)**: can be turned off; label shows Loop On / Loop Off
- **Fade Out button**: fades to silence over 2.5 s, then advances to next item (waits for Space/Play)
- Cues are invisible in Rehearsal, Show, and Auto modes; visible only in **Show W/Cues**
- In sidebar, cue items appear in italic teal with a ♪ suffix to distinguish them from songs

### Playlists
- **All Songs** is the default view (all songs, not deletable)
- Create named playlists via **+ New Playlist** button at the bottom of the sidebar
- Switch playlists via dropdown selector at top of sidebar
- Edit/rename: ✎ button; Delete: ✕ button (hidden for All Songs)
- Song selection uses checkboxes + drag-to-reorder (HTML5 + touch drag for mobile)
- **Playlist Progress** strip: white text below the song title showing `Playlist Progress  x:xx / x:xx` (elapsed time through playlist / total runtime). The coloured thermometer bar was removed in v2.11 — text only.
- Duration prefetch runs in background on load for accurate totals immediately

### Per-playlist per-song saved settings
- Tempo, pitch, vocal level saved **per song per playlist**
- **Explicit save** — press the **💾 Save settings** button in the player to store current values
- Visual dot indicator on songs that have saved settings in the active playlist
- On selecting a song: defaults applied first, then saved settings overlaid if present
- **Clear** button to remove saved settings for a song
- Stored in `localStorage` under key `stemplayer_playlist_settings`
- Export/Import carries settings with it

### Session memory (v2.11)
- In-session setting changes are remembered even without pressing Save
- Three-layer restore on song select: defaults → session memory → saved playlist settings
- Switching away and back to a song restores whatever you had dialled in that session
- Session memory resets when the page is reloaded (intentional — saved settings are the persistent layer)

### Settings indicator (v2.11)
- Orange text line below the song title when tempo/pitch/vocal differ from defaults
- Format: `⚙  85% tempo  ·  +2 semitones  ·  vocals 50%`
- Disappears when all settings are at default values
- Updates live as sliders move

### Loop region (v2.15)
- Single bar with two draggable dot markers: **green = IN**, **red = OUT**
- Drag either dot to set the loop boundary; click anywhere on the bar to snap the nearest dot to that position
- Region between dots glows green; outside is dark
- Loop button (⟳) loops within the region with a 2-second pre-roll
- ✕ Clear button resets to full track
- Only visible in Rehearsal mode

### Other
- Keyboard shortcuts: `Space` play/pause, `←/→` ±5s seek, `Escape` stop+rewind, `N` next song, `C`/`G` vamp controls
- Version badge visible in header (e.g. `v2.10`)
- Mobile-responsive: sidebar slides in as drawer on narrow screens
- Offline/cached badge — service worker (`sw.js`) handles caching

## Workflow: Adding New Songs

1. Drop new MP3 pairs into the `audio/` folder
   - Naming convention: `NN - Song Title - FINAL (Instrumental).mp3` and `(Vocals).mp3`
   - For vamp songs: `NN - Song Title PT1 - ... (Instrumental).mp3`, `VAMP`, `PT3`
2. Run `python3 generate_songs.py` from the repo root
3. Commit and push `songs.json` (and the new MP3s)
4. Bump `sw.js` version to force browsers to re-fetch the updated `songs.json`

## Workflow: Regenerating songs.json from the Word Script (sync_script.py)

Use this when the script is the source of truth — songs and cues appear in the order they occur in the show.

```
python3 sync_script.py "path/to/script.docx"
python3 sync_script.py "path/to/script.docx" --dry-run   # report only, no write
```

**Word doc setup:**
- Songs: paragraph style named exactly `song title`
- Cues: character style named exactly `Cue Title` applied inline in stage directions

**How it works:**
- Walks the script in order; each `song title` paragraph → song entry, each `Cue Title` run → cue entry
- Consecutive `Cue Title` runs in the same paragraph are merged (Word splits styled text into multiple XML runs internally)
- Matches names against audio files using `clean_name()` normalisation (strips track prefixes, FINAL, stems, smart quotes/dashes)
- Falls back to fuzzy matching (`difflib.SequenceMatcher`, threshold 0.82) for near-misses
- Duplicate cue names (same cue appearing multiple times) auto-suffix: `Wand`, `Wand (2)`, `Wand (3)`
- Version numbers are preserved in names — `It's You - Version 1` and `It's You - Version 2` are distinct

**Backups:** Previous `songs.json` is saved to `versions/songs-pre-sync.json` before overwriting.

**Dependencies:** `pip3 install python-docx`

## Workflow: Adding Music Cues

1. Drop cue MP3 into the `audio/` folder
   - Naming convention: `NNa - Cue Title (Cue).mp3` (e.g. `01a - Scene Change (Cue).mp3`)
   - Use alphanumeric prefix so the cue sorts between its surrounding songs (`01`, `01a`, `01b`, `02`)
2. Run `python3 generate_songs.py` — script auto-detects `(Cue)` suffix
3. Upload updated `songs.json` and the new MP3 to GitHub
4. Bump `sw.js` version

## Key Design Decisions

- **Single HTML file**: Everything (SoundTouch, styles, app logic) lives in `index.html`. No build step, no dependencies, no CDN required. Deployable anywhere including GitHub Pages.
- **`file://` compatible**: Intentionally avoids AudioWorklet (which requires a server). Works when opened directly from disk.
- **audio/ subfolder**: MP3s live in `audio/` so `index.html` and `songs.json` stay at the repo root, matching GitHub Pages conventions.
- **songs.json as manifest**: Decouples the player from the file system. Regenerated by script; never hand-edited.
- **No framework**: Vanilla JS only. Keeps it auditable and dependency-free.
- **Explicit settings save**: Per-playlist settings require a deliberate Save button press — prevents accidental saves while auditioning tempo changes.
